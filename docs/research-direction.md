# Research Direction Fit

## Confirmed Research Direction

AI-assisted cybersecurity investigation and threat intelligence analysis for identifying attack patterns and improving incident response.

## How This Project Supports the Direction

This project supports my research direction by demonstrating how observable phishing indicators can be extracted, scored, and explained in a structured investigation workflow. The current rule-based pipeline focuses on sender traits, social engineering language, suspicious links, attachment risk, and high-risk user prompts.

The project is intentionally explainable. Each detection result can be traced back to specific indicators, which makes it useful as a baseline before adding AI-assisted classification, summarization, or incident triage. This is important for cybersecurity investigation because analysts need outputs that are not only automated, but also understandable and reviewable.

The project also creates a starting point for future AI-assisted workflows. A next version could use AI to summarize suspicious emails, group related indicators, map observed behavior to MITRE ATT&CK techniques, assign an initial risk level, and generate recommended response actions.

## Portfolio Value

For graduate school applications, this project shows that I can connect language analysis, phishing behavior, and security investigation logic. It also shows a practical path from a simple rule-based detector toward a broader AI-assisted cybersecurity investigation system.

## Next Development Step

The next related portfolio project should expand from phishing email scoring into AI-assisted incident triage. It could accept inputs such as suspicious email text, URLs, login events, or threat reports, then produce:

- Extracted indicators of compromise
- Attack pattern summary
- MITRE ATT&CK mapping
- Initial risk score
- Suggested incident response actions
